## 3.2、系统备份
使用WinPE/启动U盘运行Ghost做镜像备份，流程如下：
一、准备启动盘与存储
- 下载含 Ghost 的启动环境（WinPE 或 Hiren’s BootCD PE）。
- 制作启动U盘，见[[#2、制作U盘启动盘]]
- 准备外置硬盘/大U盘作为镜像存放位置，格式为 NTFS。
二、系统前置设置（Windows 内）
- 暂停 BitLocker（如启用）：控制面板→BitLocker→暂停。
- 关闭“快速启动”：电源选项→选择电源按钮功能→取消勾选“启用快速启动”。
- 关机后插入启动U盘与外置盘。
三、进入 Ghost（离线）
- 从U盘启动进 WinPE/Hiren’s PE。
- 运行 Ghost（ghost32/ghost64.exe）。
四、选择镜像方式与源目标
- 整盘镜像（推荐，确保可还原启动）：Local → Disk → To Image。
	 - 源磁盘：选系统盘（通常是 Disk 1）。
	 - 目标：选择外置盘路径，命名备份文件（.gho/.ghs）。
- 若仅备份系统分区：Local → Partition → To Image，选择 C:，但需另备份EFI/System Reserved分区以保证可启动。
五、参数与选项（视版本而定）
- 压缩：高压缩（z9）体积更小，速度稍慢。
- 分卷：每2GB或4GB切分，便于拷贝到FAT32介质（NTFS可不分）。
- 校验：启用 CRC/校验，备份完成后立即“Check Image”验证。
- 跳过坏扇区：保持默认智能跳过；如加密或特殊分区，用“sector-by-sector”。
六、执行备份与校验
- 开始备份，完成后在 Ghost 里执行“Validate/Check Image”。
- 退出后重启到Windows，确认外置盘上镜像可读。
老机型XPS L502x注意
- 多为Legacy BIOS + MBR：优先整盘镜像，确保MBR与“System Reserved”分区被备份/还原。
- 若你后来改成GPT/UEFI（改BIOS或改盘）：需同时备份/还原 EFI System Partition（ESP）与 MSR 分区，否则不启动。
- 机械盘（7200rpm）：备份时间较长，确保供电稳定；散热不佳易掉速，建议在冷启动后执行。
故障与兼容建议
- Ghost版本过老对4K高级格式/UEFI支持不佳：优先在WinPE中使用较新的Ghost或考虑替代（如Clonezilla/企业版Ghost Solution Suite）。加密卷或特殊分区选“sector-by-sector”。
- 校验失败或镜像不可读：重新备份并启用CRC；外置盘用优质线缆与直连端口。
## 3.3、系统还原（灾难恢复）
流程如下：
- 进WinPE运行 Ghost。
- 还原整盘：Local → Disk → From Image（会覆盖整盘，包含分区表/引导）。
- 若换盘（如从HDD到SSD）：先用磁盘管理或Ghost的“From Image”做整盘还原，确保对齐；Windows首次启动会自适配驱动。必要时用 bcdboot 修复引导。

故障与兼容建议
- 还原后无法启动：
	- MBR/Legacy：用安装介质修复引导→命令提示符运行 bootrec /fixmbr /fixboot /rebuildbcd。
	- UEFI/GPT：在WinRE/PE里运行 bcdboot C:\Windows /l zh-cn /s X: /f UEFI（X为ESP盘符）。

## 4.2、WSL Ubuntu
### 4.2.1、命令行安装

> [!NOTE] 先决条件
> Windows”启用或关闭Windows功能”里勾选“适用于Linux的Windows子系统”和“虚拟机平台”，然后重启。

```powershell
# 1. 启用并设置WSL
wsl --install
# 若已启用WSL但未装发行版：
wsl --set-default-version 2
wsl --update

# 2. 安装Ubuntu发行版
## 直接从最新官方Ubuntu安装
wsl --install -d Ubuntu
## 指定版本（例如 Ubuntu-20.04 或 Ubuntu-22.04）
wsl --install -d Ubuntu-20.04
# 或
wsl --install -d Ubuntu-22.04

# 3. 安装完成后终端会提示创建Linux用户名与密码；按提示完成初始化。

```
